# 🧭 Chrome 133 新用語リファレンス（Glossary）

> 本Glossaryは **docs/chrome-133-2025-02-04.md** の「全体サマリ」の直後に挿入予定の内容です（PR前の準備版）。

| 用語 | 概要 | 関連領域 |
|------|------|-----------|
| **Baseline** | 主要ブラウザ横断で“安定して使える”機能セットの合意点。設計時の対応範囲を明確化できる。 | 互換性 / 企画 |
| **Origin Trial (OT)** | 期限付きトークンで新機能を早期評価できる仕組み。APIや挙動が変更される可能性がある。 | 実験機能 / 運用 |
| **View Transitions** | ページ遷移やDOM更新時のスムーズなトランジションを実装する仕組み。UX向上に寄与。 | CSS / UX |
| **Popover API** | 補助的UI（メニュー・ツールチップ等）を標準方式で実装するAPI。フォーカス管理や外側クリック閉じを内包。 | HTML / アクセシビリティ |
| **Navigation API** | SPA的遷移を標準APIで実現し、履歴や遷移フックを提供。 | ルーティング / パフォーマンス |
| **PII混入** | 個人特定情報がログやプロンプトに混ざること。収集・保管ポリシーとマスキング設計が必要。 | プライバシー |
| **SRI/CSP** | Subresource Integrity と Content Security Policy。改ざん検出と読み込み制御で供給網リスクを軽減。 | セキュリティ |
